Q: What is Object-Oriented Programming (OOP)?

OOP is a programming paradigm that organizes software design around objects rather than functions and logic.

4 Pillars of OOP:
  1. Inheritance: Child class inherits properties from parent class
  2. Polymorphism: One name, multiple forms
  3. Abstraction: Hiding implementation details
  4. Encapsulation: Wrapping data and methods together
// Simple OOP Example
class Car {
    // Attributes
    String color;
    String model;
    
    // Methods
    void start() {
        System.out.println("Car starting...");
    }
    
    void drive() {
        System.out.println("Driving " + model);
    }
}

Real-world Analogy:

  • Object: A specific car (like your Honda Civic)
  • Class: Blueprint of a car
  • Attributes: Color, model, engine type
  • Methods: Start, drive, brake, stop
Q: What are Classes and Objects in Java?

Class: Blueprint or template for creating objects.

Object: Instance of a class with state and behavior.

Key Points:
  • Class exists in code, Object exists in memory at runtime
  • Use new keyword to create objects
  • Multiple objects can be created from one class
  • Each object has separate memory space
// Class Definition
class Student {
    // Attributes (State)
    String name;
    int age;
    String course;
    
    // Methods (Behavior)
    void study() {
        System.out.println(name + " is studying " + course);
    }
    
    void displayInfo() {
        System.out.println("Name: " + name + ", Age: " + age);
    }
}

// Creating Objects
public class Main {
    public static void main(String[] args) {
        // Object 1
        Student student1 = new Student();
        student1.name = "Aman";
        student1.age = 20;
        student1.course = "Java";
        student1.study();
        
        // Object 2
        Student student2 = new Student();
        student2.name = "Rohan";
        student2.age = 22;
        student2.course = "Python";
        student2.displayInfo();
    }
}

Output:

Aman is studying Java
Name: Rohan, Age: 22
Q: What is Inheritance in Java?

Inheritance allows a class to acquire properties and methods of another class.

Benefits:
  • Code Reusability: Reuse parent class code
  • Method Overriding: Runtime polymorphism
  • Extensibility: Easy to extend functionality
// Parent Class
class Vehicle {
    String brand;
    int year;
    
    void start() {
        System.out.println("Vehicle starting...");
    }
    
    void displayInfo() {
        System.out.println("Brand: " + brand + ", Year: " + year);
    }
}

// Child Class
class Car extends Vehicle {
    int doors = 4;
    
    void honk() {
        System.out.println("Beep Beep!");
    }
}

// Another Child Class
class Bike extends Vehicle {
    boolean hasCarrier = true;
    
    void wheelie() {
        System.out.println("Doing a wheelie!");
    }
}

// Main Class
public class Main {
    public static void main(String[] args) {
        Car myCar = new Car();
        myCar.brand = "Toyota";  // Inherited from Vehicle
        myCar.year = 2023;       // Inherited from Vehicle
        myCar.start();           // Inherited from Vehicle
        myCar.honk();            // Own method
        
        Bike myBike = new Bike();
        myBike.start();          // Inherited from Vehicle
        myBike.wheelie();        // Own method
    }
}

Types of Inheritance:

  • Single: One child, one parent
  • Multilevel: A extends B, B extends C
  • Hierarchical: Multiple children, one parent
  • Multiple: Not in Java (use interfaces)
